# Impeccable Design Context

## Design Context

### Users
Developers and ops engineers who build and operate agentic AI workflows. They interact with the dashboard to define pipelines, monitor runs, manage skills and packages, review approvals, and track costs. They are technically sophisticated, comfortable with YAML, cron expressions, and API concepts. They value efficiency and clarity over hand-holding. Their primary job: orchestrate AI agent workflows reliably and watch them execute in real-time.

### Brand Personality
**Confident, precise, operational.** Pawn is a control surface for autonomous systems -- it should feel like a mission control dashboard, not a consumer app. The tone is direct and understated; the interface earns trust through clarity and competence, not decoration.

### Aesthetic Direction
- **Dark mode only** -- slate-950 base, slate-900 cards, slate-800 borders
- **Accent**: sky-400/sky-500 primary, emerald for success, amber for warnings, rose for errors
- **Typography**: Satoshi (body), Cabinet Grotesk (display/headings). Tight tracking on headings.
- **Density**: Information-dense but breathable. Small text (11px-14px) with generous padding on containers.
- **Shape language**: Rounded (rounded-xl/2xl cards, rounded-md badges), soft glows over hard shadows
- **Animation**: Subtle and purposeful -- fade-in-up for content, scale-in for modals. Respects prefers-reduced-motion.
- **Anti-references**: Colorful SaaS marketing dashboards, playful/whimsical UIs, heavy illustrations, chatbot-style interfaces. This is NOT Notion or Linear's marketing site -- it is closer to Grafana or Datadog in spirit (but with better typography and calmer density).

### Design Principles
1. **Earned trust**: Every element should reinforce that the system is working correctly. Status indicators, timestamps, and feedback are never ambiguous.
2. **Progressive density**: Start sparse for new users, allow information density to grow as the user populates the system. Empty states bridge the gap.
3. **Contextual guidance**: Teach at the point of need, not upfront. Empty states, tooltips, and inline hints over onboarding wizards.
4. **Quiet confidence**: Restrained color, no unnecessary motion, no exclamation marks. The UI is calm even when showing errors.
5. **Operational clarity**: Every screen answers "what is happening right now?" before "what could I do?" Status is always visible and unambiguous.

### Component Patterns
- **Cards**: `bg-slate-900/50 border border-slate-800 rounded-2xl`
- **Buttons (primary)**: `bg-sky-500 hover:bg-sky-400 text-slate-950 rounded-xl text-xs font-bold`
- **Buttons (secondary)**: `bg-slate-800 hover:bg-slate-700 text-slate-300 rounded-md`
- **Badges**: Tinted background + matching text + subtle border (e.g., `bg-sky-500/10 text-sky-400 border border-sky-500/20`)
- **Section headers**: `text-[11px] font-bold text-slate-400 uppercase tracking-widest`
- **Page headers**: `text-2xl font-bold font-display text-white tracking-tight`
- **Empty states (current)**: Minimal text + link. These are the primary target for improvement.

### Icon System
Lucide React icons throughout. 12-16px standard size. Icons use contextual accent colors or slate-400/500 for neutral states.
